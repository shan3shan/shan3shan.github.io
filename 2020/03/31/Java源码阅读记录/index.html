ArrayList

  • 数组实现，默认大小10
  • 扩容规则：int newCapacity = oldCapacity + (oldCapacity >> 1);
  • 扩容时机：add前进行检查，不够时扩容。、
  • fastRemove()相较remove()减少了删除范围的检测以及删除后的返回值。
  • 有内部类Itr实现迭代器接口，Itr和ArrayList都有自己的remove和add方法，混用可能回触发fail-fast机制（常见的再foreach中删除增加元素），导致接口报错。

LinkedList

  • 双向链表实现
  • 非线程安全
  • 可以在定义创建时设置为同步的：List list = Collections.synchronizedList(new LinkedList(…))
  • 提供listIterator
  • 提供spliterator（并行遍历迭代器）

HashMap

  • 允许空值空键
  • 和Hash Table类似除了它线程不安全以及支持空值
  • 要保证迭代能力，就要尽量保证负载因子不要太高（初始容量设置太大）.
  • 有两个影响性能的原因：负载因子（load factor）和初始容量（initial capacity）
    • initial capacity：is the number of buckets in the hash table, and the initial capacity is simply the capacity at the time the hash table is created.
    • load factor：is a measure of how full the hash table is allowed to get before its capacity is automatically increased.
    • 当容量大于这两项的乘积时，进行rehash。
    • 默认的负载因子 0.75，初始容量1<<4，最大容量1<<30.
    • 可以在创建时提供参数设置初始容量和负载因子。
  • 迭代使用Iterator，有fail-fast机制。
  • resize方法：oldcap左移一位。二倍。

HashTable

  • 键值不能为空。
  • 作为键的对象必须实现hashcode()和equals()方法。
  • 负载因子0.75，默认大小11.
  • 若使用Enumration 枚举，没有快速失败机制。
  • 是同步的，关键方法都使用synchhronized
  • 提供的Enumerator实现了Enumeration和Iterator接口，因此存在快速失败机制。

HashSet

  • 由HashMap实现，有key存储元素。
  • 可以传空值。

LinkedHashMap

  • 增加一个双向链表来保证顺序。

ConcurrentHashMap

  • 1.7使用分段锁，1.8使用CAS和 synchronized

  • 先了解两个概念：

    • volatile：可见性，修饰变量可以保证变量在被修改时能第一时间更新到内存上，从而保证所有线程都能获取到最新的值。 synchronized 和lock也能保证可见性。

    • CAS：Compare And Swap 比较并交换。

      • 是一种无锁算法，一般有三个操作数，内存值，旧的预期值，要改的新值。

      • 仅当内存值和旧的预期值相同时，才将内存值改为 新值。

      • do{

        备份旧数据；

        基于旧数据构造新数据；

        }while(!CAS( 内存地址，备份的旧数据，新数据 ))

  • Node<k,v>,TreeNode<k,v>都使用了volatile修饰，确保看到的是最新的值。
